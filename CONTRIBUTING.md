# Contributing to E-ZPassFL


Thank you for your interest in contributing to **E-ZPassFL**! We welcome contributions of all kinds — code, documentation, bug reports, and feature suggestions.


## Contribution Guidelines


- Be respectful and considerate when interacting with maintainers and contributors.
- Keep changes small and focused; prefer multiple small PRs to one large PR.
- Write clear commit messages. Use present-tense, short subject line and optional longer body.
- Follow existing code style and naming conventions.


## How to Contribute


1. Fork the repository.
2. Create a new branch: `git checkout -b feature/your-feature-name`.
3. Make your changes in the branch.
4. Keep commits organized and atomic.
5. Run tests and linters before committing.
6. Push your branch to your fork and open a Pull Request against `main`.


## Reporting Issues


- Please open issues for bugs or unexpected behavior.
- When reporting, include steps to reproduce, expected behavior, actual behavior, and screenshots if helpful.
- Tag the issue with an appropriate label when possible (bug, enhancement, question).


## Suggesting Features


- Open a new issue with the `enhancement` label.
- Describe the problem the new feature solves, provide examples, and (optionally) propose an API or UI mock.


## Code of Conduct


This project follows a Code of Conduct to ensure a welcoming environment for all contributors. Treat others with respect. Harassment, hate speech, and discriminatory language will not be tolerated.


## Review Process


- Pull requests are reviewed by the maintainers.
- Expect review comments requesting changes or clarification — this helps maintain code quality.
- Major or architectural changes may require design discussions before implementation.
- Once approved, a maintainer will merge your PR.


Thanks for helping make E-ZPassFL better!
